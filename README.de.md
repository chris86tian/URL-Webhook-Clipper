# 🚀 URL Webhook Clipper – Die beste Webhook Chrome-Erweiterung
## 🌟 Automatisiere deine Workflows mit Webhooks in nur einem Klick!

**URL Webhook Clipper** ist eine leistungsstarke **Webhook Chrome-Erweiterung**, mit der du **URLs, Notizen und Dateien direkt an mehrere Webhook-Endpunkte und Airtable senden kannst**.
Perfekt für die Automatisierung von Workflows mit **Make.com, Zapier, N8N, Slack, ClickUp, Notion, Airtable, Google Sheets und CRM-Tools** wie **HubSpot, Pipedrive und Salesforce**.

## 🎯 So funktioniert es – Webhooks in 3 Schritten automatisieren

### Standard-Workflow

1. **Klicke auf das Erweiterungssymbol** in deiner Chrome-Symbolleiste
2. **Wähle dein Ziel** (Webhook oder Airtable-Tabelle)
3. **Sende deine Daten** – URLs, Notizen, Dateien und mehr!

### Rechtsklick-Workflow (NEU in v2.0)

1. **Rechtsklick** auf eine Webseite, Link, Bild oder markierten Text
2. **Bewege die Maus** über "Send to Webhook/Airtable"
3. **Wähle dein Ziel** aus dem Untermenü (🔗 Webhooks / 📊 Airtable)
4. **Inhalt wird automatisch gesendet**
5. **Erhalte Erfolgs-/Fehler-Benachrichtigung**

📌 **Perfekt für Lead-Erfassung, Aufgabenautomatisierung und KI-gestützte Workflows.**

## 🆕 Kontextmenü-Integration (Version 2.0)

### Wie funktioniert das Rechtsklick-Menü?

- **Dynamisches Menü**: Aktualisiert sich automatisch, wenn Ziele hinzugefügt/entfernt werden
- **Gruppierte Ziele**: Webhooks und Airtable-Tabellen in separaten Untermenüs
- **Intelligente Payload-Erstellung**: Extrahiert automatisch URL, Titel, Auswahl und Meta-Beschreibung
- **Benachrichtigungen**: Zeigt Erfolg oder Fehler nach dem Senden an
- **Kontexte**: Funktioniert auf Seiten, Links, Bildern und markiertem Text

### Automatische Datenerfassung

Das Kontextmenü erfasst automatisch:
- **URL**: Link-URL, Bild-URL oder aktuelle Tab-URL
- **Titel**: Seitentitel
- **Notizen**: Markierter Text (falls vorhanden)
- **Meta-Beschreibung**: Seitenbeschreibung (falls verfügbar)
- **Zeitstempel**: Aktuelles Datum und Uhrzeit
- **Kontext-Typ**: Art des Rechtsklicks (Seite, Link, Bild, Auswahl)

## 🆕 Session-Persistenz (Version 1.7+)

### Wie funktioniert die Persistenz?

- **Auto-Save**: Formulardaten werden automatisch beim Tippen gespeichert (500ms Verzögerung)
- **Persistente Felder**: Notizen, Ziel-Auswahl, Template-Auswahl und Dateianhänge
- **Session-Only**: Daten bleiben nur während der Browser-Session erhalten (werden beim Browser-Neustart gelöscht)
- **Auto-Clear**: Daten werden automatisch nach erfolgreichem Senden oder bei Fehlern gelöscht
- **Manuelles Löschen**: Nutze den "Clear Form" Button zum schnellen Zurücksetzen
- **Dynamische URL**: Die aktuelle Tab-URL wird immer frisch beim Senden abgerufen (nie gespeichert)

## 🔧 Webhook Payload & JSON-Format

Die Erweiterung sendet Daten im folgenden JSON-Format:

```json
{
  "url": "https://example.com",
  "title": "Beispiel-Seitentitel",
  "notes": "Benutzer eingegebene Notizen",
  "template": "Lead-Erfassung",
  "metaDescription": "Meta-Beschreibung der Seite",
  "timestamp": "2024-03-13T20:00:00.000Z",
  "contextType": "page",
  "attachments": [
    {
      "name": "dokument.pdf",
      "type": "application/pdf",
      "data": "base64-encoded-content"
    }
  ]
}
```

## 🏗️ Modulare Architektur (Version 1.8+)

### Dateistruktur

```
URL-Webhook-Clipper/
├── manifest.json           # Extension manifest (v2.0)
├── background.js          # Background service worker mit Kontextmenü
├── popup/
│   ├── popup.html        # Haupt-UI
│   ├── popup.js          # Haupt-Orchestrierung
│   ├── styles.css        # Alle Styles (Apple-Style)
│   └── modules/          # Modulare Architektur
│       ├── storage.js    # Session Storage Management
│       ├── theme.js      # Dark Mode Handling
│       ├── fileHandler.js # Datei-Anhang Logik
│       ├── sender.js     # Webhook/Airtable-Versand Logik
│       ├── webhookManager.js # Webhook CRUD Operationen
│       ├── airtableManager.js # Airtable CRUD Operationen
│       └── airtableSender.js # Airtable API Integration
└── icons/
```

### Module im Detail

- **storage.js**: Session Storage Operationen
  - Formulardaten speichern/laden/löschen
  - Aktuellen Formular-Status abrufen
  - Formular-Status in UI wiederherstellen

- **theme.js**: Dark Mode Management
  - Theme basierend auf Präferenz initialisieren
  - Theme umschalten
  - Theme-Icons aktualisieren

- **fileHandler.js**: Datei-Anhänge verarbeiten
  - Drag & Drop Handling
  - Datei-Validierung
  - Anhang-Management

- **sender.js**: Daten an Webhooks/Airtable senden
  - Aktuelle Tab-Info abrufen
  - Payload vorbereiten
  - Webhook/Airtable-Antworten verarbeiten

- **webhookManager.js**: Webhook-Konfigurationen verwalten
  - CRUD-Operationen für Webhooks
  - Template-Management
  - Import/Export-Funktionalität

- **airtableManager.js**: Airtable-Konfigurationen verwalten
  - CRUD-Operationen für Airtable Bases
  - Tabellen-Schema abrufen
  - Field-Mapping-Konfiguration

- **airtableSender.js**: Airtable API Kommunikation
  - Record-Erstellung
  - Field-Type-Handling
  - Collaborator-Unterstützung

### Background Service Worker (NEU in v2.0)

- **Dynamisches Menü-Building**: Kontextmenü aktualisiert sich automatisch bei Änderungen
- **Gruppierte Ziele**: Webhooks und Airtable-Tabellen in Untermenüs organisiert
- **Intelligente Payload-Erstellung**: Extrahiert automatisch URL, Titel, Auswahl und Meta-Beschreibung
- **Fehlerbehandlung**: Zeigt Benachrichtigungen für Erfolg/Fehler
- **Storage-Monitoring**: Hört auf Config-Änderungen und baut Menü neu

## 🔒 Datenschutz & DSGVO-Konformität

- ✅ **Keine Tracking-Tools, keine Analyse-Daten** – 100 % datenschutzfreundlich
- ✅ **Alle Daten bleiben lokal gespeichert** – Konfigurationen verlassen niemals deinen Browser
- ✅ **Session-Only Persistenz** – Formulardaten werden nur während der Browser-Session gespeichert
- ✅ **Nur Daten senden, die du explizit auswählst** – Volle Kontrolle über deine Daten

### 📌 So löschst du gespeicherte Daten

1. **Formulardaten löschen**: Nutze den "Clear Form" Button
2. **Webhook-Konfigurationen entfernen**: Lösche Webhooks in den Einstellungen
3. **Airtable-Konfigurationen entfernen**: Lösche Airtable Bases in den Einstellungen
4. **Browser-Speicher löschen**: Chrome-Einstellungen → Datenschutz → Browserdaten löschen
5. **Erweiterung deinstallieren**: Entfernt alle gespeicherten Daten
